## 📋 CODING STANDARDS

### Git Workflow
```
1. Create branch from main
2. Make changes
3. Commit with descriptive message
4. Push to remote
5. Create Pull Request
6. Code review
7. Merge to main
```

### Commit Message Format
```
<type>(<scope>): <description>

Types: feat, fix, docs, style, refactor, test, chore
```
